# Security Policy

CERBERUS Cyber is a defensive security research project. Please report vulnerabilities responsibly.

## Reporting a Vulnerability

Do not open a public issue for a vulnerability that could expose users, credentials, infrastructure, or sensitive data.

Instead, use GitHub's private vulnerability reporting feature when available, or contact the repository owner privately.

Include:

- affected component
- reproduction conditions
- expected and observed behavior
- likely impact
- suggested mitigation, if known

## Scope

Security reports are especially welcome for:

- policy bypasses
- unsafe action authorization
- privilege escalation
- connector impersonation
- secret leakage
- prompt injection affecting tool use
- evidence tampering
- rollback or recovery failures

## Defensive-Only Boundary

Reports and contributions must not include harmful payloads, stolen data, unauthorized access, or instructions for attacking third-party systems.
